## Key Differences

| Concept | Tussel/Strudel | Tidal |
|---------|---------------|-------|
| Method chaining | `.fast(2).slow(2)` | `fast 2 $ slow 2 $` |
| Properties | `.gain(0.5).pan(0.3)` | `# gain 0.5 # pan 0.3` |
| Lambda functions | `x => x.fast(2)` | `(fast 2)` (partial application) |
| Multi-channel | `stack(...)` or `scene({channels})` | `d1 $`, `d2 $`, etc. |
| Tempo | `setcps(0.75)` | `setcps 0.75` |

Tussel and Strudel share nearly identical syntax for most operations. The main difference from Tidal is the switch from Haskell's function application style to JavaScript/TypeScript method chaining.
