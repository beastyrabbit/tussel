`@tussel/cli` provides the command-line interface. It's a thin wrapper around `@tussel/runtime` functions.

## Entry Point

```ts
async function main(argv?: string[]): Promise<void>
```

Parses command-line arguments and dispatches to the appropriate command handler.

## Commands

| Command | Runtime Function | Description |
|---------|-----------------|-------------|
| `check <file>` | `checkScene()` | Validate a scene file |
| `run <file>` | `runScene()` | Play a scene with the daemon |
| `render <file>` | `renderScene()` | Render to WAV |
| `convert <file>` | `convertScene()` | Convert between formats |

## Usage

```bash
# From the repository
pnpm exec tussel <command> [options]

# Or via the bin entry
./bin/tussel <command> [options]
```

See [CLI Reference](/tussel/docs/cli-reference/) for full option documentation.
